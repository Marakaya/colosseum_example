# Contributing to BBM

Thank you for your interest in contributing to Block Building Mechanism!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest features
- Include steps to reproduce, expected and actual behavior
- Attach logs or screenshots where relevant

### Submitting Pull Requests
1. Fork the repository
2. Create a feature branch: `git checkout -b feat/your-feature`
3. Make your changes with clear, conventional commit messages
4. Ensure tests pass: `anchor test`
5. Open a Pull Request against `main`

### Commit Convention
We use [Conventional Commits](https://www.conventionalcommits.org/):
```
feat: add new bundle type
fix: resolve conflict detection edge case
docs: update architecture diagram
test: add unit tests for relay
```

### Development Setup
See [Quick Start](README.md#quick-start) in the README.

## Code of Conduct
Be respectful and constructive. We're building in public on Solana — let's keep it collaborative.
